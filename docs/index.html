Opportunity Scoring

A data-driven model for identifying visible pages that under-capture clicks, enabling prioritized content review with transparent reason codes.

Ojas Saini · July 2026 · Lane: Content Refresh / Decay
Abstract

Which existing content pages should receive a refresh review to recapture declining search visibility? We train a Random Forest classifier on 30,000 anonymized content items from the FlyRank warehouse to predict decline risk, achieving 0.750 ROC AUC and 74.0% Precision@50 on a held-out client split — a 3× lift over the hand-coded baseline (24.0% Precision@50). The final ranked queue assigns transparent action codes (refresh, refresh + review CTR, expand + refresh, monitor) to each page, giving editors a prioritized, explainable starting point for content review decisions.

01Introduction

Content teams managing hundreds or thousands of published pages face a recurring decision: which pages deserve a refresh review this cycle? Manual triage is slow, and blanket staleness rules (e.g. "review anything older than 6 months") generate noisy queues that overwhelm editors.

This project frames the problem as a supervised classification task: given a page's search performance, engagement metrics, content properties, and freshness indicators, predict whether it is experiencing a decline trend — the signal that a refresh review is warranted.

The output is a ranked refresh queue with reason codes that explain why each page appears, supporting informed editorial judgment rather than replacing it. The model is a decision-support tool, not an automatic publishing decision.

The Decision

"Should this page receive a content refresh review?"

  • Unit of analysis: one content page
  • Output: priority score + action code
  • Human action: manual review & editorial update
  • Cost of error: wasted editor time (false positive) or missed decay (false negative)

Baseline Formula

baseline_score =
    0.40 × visibility_score
  + 0.30 × freshness_risk_score
  + 0.25 × position_opportunity_score
  + 0.05 × depth_gap_score

The baseline uses only percentile ranks and hand-tuned weights — no machine learning. It serves as the "transparent rule" that the model must demonstrably beat on the same test split.

Interpretation

The Random Forest model's top features are search visibility metrics — days with impressions, impression volume, and average position account for the top three importance slots. This is intuitive: pages that have been consistently visible and are slipping in rankings are the strongest candidates for refresh review.

Content properties (age, word count, character count) form the second tier, confirming that older and thinner content is directionally associated with higher decline risk.

The 3× Precision@50 lift over baseline (74% vs 24%) demonstrates that the model captures non-linear feature interactions that simple weighted rules miss — particularly the interplay between position, volume, and content freshness.

The near-balanced label distribution (54.2% declining) means the base rate is high; a random ranker would achieve ~54% Precision@50. The model's 74% represents meaningful lift above both random and the rule-based baseline.

05Limitations

⚠

Observational, Not Causal

This is observational data. We measure association between features and decline, not causation. "Associated with decline" does not mean "causes decline."

⚠

Label Is a Proxy

The label (trend_direction == "down") captures 30-day impression trends, not the ground truth of "needs refresh." Some declining pages may be intentionally deprecated; some stable pages may still benefit from updates.

⚠

Starter Dataset Only

Results are from the 30K-row anonymized starter dataset. The full warehouse (~79M daily rows) may reveal different patterns at scale. Model should be re-validated on the full release.

⚠

Single Time Snapshot

Training and test data come from the same trailing 90-day window. Temporal stability (does the model hold month-over-month?) is untested and should be validated before production deployment.

⚠

No Content Text

The model uses only structured metadata and aggregated metrics — no actual page content, titles, or semantic analysis. A richer model could use NLP features for deeper quality signals.

⚠

Decision-Support Only

Model output is a prioritization aid for human reviewers. It should never be used as an automatic publishing or unpublishing decision without editorial verification.

06Ranked Recommendations

The queue assigns each page one of five action codes based on model probability, baseline signals, and engagement patterns. Start with high-confidence rows.

refresh_and_review_ctr — High impressions, low CTR, declining. Review title/meta.
refresh_and_review_engagement — Low engagement/scroll despite traffic. Review content quality.
refresh — Declining or stale; general content update recommended.
expand_and_refresh — Thin content (<1,200 words) with visibility. Expand depth.
monitor — No urgent signals; re-evaluate next cycle.
